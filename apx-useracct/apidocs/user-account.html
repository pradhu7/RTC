Attempts to authenticate a User.

post /auths

Attempts to authenticate a User.

  • Request
  • Response

Query Parameters

  • int: (boolean )

    Means "internal". Set to true if the token returned should be an internal token.

  • ttl: (integer )

    Time-to-live for token, in seconds.

Body

Type: application/json

Schema:

{
  "email": "<email address>; required if in initial authentication (i.e., not part of 2nd factor)",
  "password": "<password>; required if in initial authentication (i.e., not part of 2nd factor)",
  "code": "<code>; required if in second step of 2nd factor"
}

HTTP status code 200

Returned on successful authentication

Body

Type: application/json

Schema:

{
  "token": "<token-value>"
}

HTTP status code 401

Returned on authentication failure. Note that this is non-conforming for 401 as it doesn't return a WWW-Authenticate header with a challenge.

Body

Type: application/json

Schema:

{
  "reason": "<failure info>"
}

/uorgs/properties get

get

Gets ALL custom properties of ALL UserOrg entities in the system. The returned JSON structure is an object with field names that are the XUUID of the UserOrg entity, and the value of the field is another JSON object where the field names of that are the custom property name and the field values are the values of the custom properties.

The returned JSON structure is an object with field names that are the XUUID of the UserOrg entity, and the value of the field is another JSON object where the field names of that are the custom property name and the field values are the values of the custom properties.

/uorgs/{entityID}/properties/{name} put delete

put

Sets (adds or replaces) a custom property of the given name ({name}) to the given UserOrg ({entityID}). The entityID must be the externalID of the UserOrg. The operation will fail if the name doesn't correspond to a known custom property (that was added by POST:/uorgs/cproperties) or if the value supplied can't be converted to the value type as declared when the custom property was defined.

delete

Deletes the given custom property ({name}) from the given UserOrg ({entityID}). Note that this only deletes the property value on that entity instance and doesn't affect the custom property definition.

/users/{entityID}/properties/{name} put delete

put

Sets (adds or replaces) a custom property of the given name ({name}) to the given User ({entityID}). The operation will fail if the name doesn't correspond to a known custom property (that was added by POST:/users/cproperties) or if the value supplied can't be converted to the value type as declared when the custom property was defined.

delete

Deletes the given custom property ({name}) from the given User ({entityID}). Note that this only deletes the property value on that entity instance and doesn't affect the custom property definition.
